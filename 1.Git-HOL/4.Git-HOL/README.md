### Understanding and Resolving Merge Conflicts in Git

## 📖 Overview

This lab demonstrates how merge conflicts occur when multiple changes are made to the same file in different branches. It covers conflict identification, conflict resolution, branch management, and updating the `.gitignore` file.

Merge conflicts are a common part of collaborative software development, and understanding how to resolve them is an essential Git skill.

---

## 🎯 Objectives

✅ Understand merge conflicts in Git

✅ Create conflicting changes in multiple branches

✅ Identify conflict markers generated by Git

✅ Resolve conflicts manually

✅ Complete a successful merge

✅ Configure `.gitignore` to exclude backup files

# 🌿 Branch Creation

## Create a New Branch

```bash
git checkout -b GitWork
```

---

## Create hello.xml

```bash
echo "<message>Hello from GitWork</message>" > hello.xml
```

Add and Commit:

```bash
git add hello.xml
git commit -m "Added hello.xml in GitWork"
```

---

## Modify hello.xml in GitWork

```xml
<message>Updated in GitWork Branch</message>
```

Commit the changes:

```bash
git add hello.xml
git commit -m "Updated hello.xml in GitWork"
```

---

# 🌳 Changes in Master Branch

Switch to master:

```bash
git checkout master
```

Create the same file with different content:

```xml
<message>Updated in Master Branch</message>
```

Commit the changes:

```bash
git add hello.xml
git commit -m "Added hello.xml in Master"
```

---

# 🔍 View Commit History

```bash
git log --oneline --graph --decorate --all
```

Example:

```text
* abc1234 (GitWork) Updated hello.xml in GitWork
| * def5678 (master) Added hello.xml in Master
|/
* 345559f Previous Commit
```

---

# ⚠️ Merge Conflict

Attempt to merge:

```bash
git merge GitWork
```

Git reports a conflict:

```text
Auto-merging hello.xml
CONFLICT (add/add): Merge conflict in hello.xml
Automatic merge failed; fix conflicts and then commit the result.
```

---

# 📝 Conflict Markers

Git inserts conflict markers into the file:

```text
<<<<<<< HEAD
<message>Updated in Master Branch</message>
=======
<message>Updated in GitWork Branch</message>
>>>>>>> GitWork
```

---

# ✅ Conflict Resolution

Resolve the conflict by updating the file:

```xml
<message>Updated in Master and GitWork Branch</message>
```

Save the file.

---

## Complete the Merge

```bash
git add hello.xml
git commit -m "Resolved merge conflict in hello.xml"
```

---

# 🚫 Updating .gitignore

Add backup files to `.gitignore`:

```text
*.log
logs/
*.bak
```

Commit the changes:

```bash
git add .gitignore
git commit -m "Added backup files to gitignore"
```

# 🌿 Branch Management

## View Available Branches

```bash
git branch
```

Example:

```text
GitWork
* master
```

---

## Delete Merged Branch

```bash
git branch -d GitWork
```

Output:

```text
Deleted branch GitWork
```
